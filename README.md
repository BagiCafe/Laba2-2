# LABA2-1 ( Источники задач и контракты)

## Установка и запуск

### Запуск Лабы
```bash
# Клонирование репозитория
git clone <https://github.com/BagiCafe/Laba2-1.git>

# Установите pytest для тестов
pip install pytest

# Запуск программы (логгирование)
python main.py
```
## Функционал

### Модель задачи (```Task```)
Простое хранилище данных задачи. Содержит только самую необходимую информацию:
- ```id``` (число) — уникальный номер задачи
- ```payload``` (словарь) — любые данные, которые нужно обработать

### Протокол источника задач (```TaskSource```)
Это договор, которому должны соответствовать все источники задач.


### Реализованные источники задач:
1) ```TaskSourceGenerator``` ( Генератор случайных задач)
   - Создаёт 10 задач со случайными данными
   - ```id```: случайное число от 1 до 100
   - ```payload```: {"user_id": случайное, "count": случайное}

2) ```TaskSourceFile``` ( Чтение из JSON-файла)
   - Читает задачи из файла
  
3) ```TaskSourceAPI``` ( Имитация внешнего API)
   - Возвращает 4 задачи с фиксированными ```id``` (10-13)
   - Данные генерируются случайно при каждом вызове
   - Имитирует сетевое взаимодействие
   
## Технические решения
- Класс ```Task``` реализуется через ```dataclass```
- Все источники реализуют метод ```get_tasks()``` ( Duck typing)
- Используется ```isinstance()``` для проверки соответствия протоколу (декоратор - ```runtime_checkable```)
- Единый контракт через typing.Protocol ( Классы ```TaskSourceGenerator```, ```TaskSourceFile```, ```TaskSourceAPI``` не наследуются от ```TaskSource```)


## Тестирование
Для запуска всех тестов необходимо ввести в консоль:
```bash
python -m pytest tests/ -v
```

Для запуска определенного теста ввидите в консоль:
```bash
python -m pytest tests/<file_name_test> -v
```
Название всех тестов:
- test_task.py
- test_sources.py
